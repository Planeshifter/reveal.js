  • Allows to define triples of 'subject', 'predicate', 'value'
  • Designed to be read and understood by computers.
  • URIs are used as names for entities.
  • HTTP URIs can be looked up to obtain useful information
  • Allows to deal with name disambiguation problem

OWL (Web Ontology Language)
  • Based on RDF, OWL allows logical statements and inference

Even though it is possible to do amazing things with computational statistical methods, computers don't understand. It is not sufficient to extract information about the co-occurence of entities, but also the nature of their relationships. Co-occurence in politics is antagonism.

Different types of text: News articles, Political Speeches

  • The latter are much more opinionated, with news articles being much more neutral
  • We can be sure that the inferred co-occurence network carries meaning for speeches, whereas the news articles require more thought.
  • To make life easier, start with the simpler problem

Problem Statement:

Inferring the sentiment (approval / disapproval) of congressional speeches towards legislation based on the documents and the implied network structure.

Literature Review:

  • Thomas, M., Pang, B., Lee, L., 2006. Get out the vote: Determining support or opposition from Congressional floor-debate transcripts. Proceedings of EMNLP.
  • Burfoot, C., Bird, S., Baldwin, T., 2011. Collective Classification of Congressional Floor-Debate Transcripts. Comput. Linguist. 1506–1515.
They do Sentiment classification of U.S. Congressional floor-debate transcripts (before a bill is voted on it is debated in chamber, where members of House and Senate discuss the merits of the bill). Their data set is available for further research. However, Library of Congress makes the congressional record available online, so that I will be able to scrape data myself too.

Collective Classification

  • Ordinary classification methods such as Logistic Regression or Naive Bayes predict the value for actor $Y_i$ using a feature vector $x_i$ containing the observed attributes of the observation
  • In collective classification, one uses also information about actors in the neighbourhood of $Y_i$ in the prediction task
  • Goal is then to infer the class labels with either the maximum joint probability or the maximum marginal probability for each node.

Classification Techniques:

  • Iterative Classifier Approach:

    One at a time, generate the best classification for a single instance given all other parts of the problem (i.e. the correct labelling of the other instances)

  • Gibbs Sampling:

    Iteratively assign class labels to all nodes by drawing from the conditional distribution of the current node given the others. After a sufficient number of iterations, count the number of times a given node was assigned a certain label and normalize to get probability estimates

  • Variational Methods:
    • Loopy Belief Propagation Loopy
    • Mean-Field Relaxation Labeling (MF)
    • Methods have in common that a trial distribution is chosen which is in some sense comparable to the true distribution (e.g. in terms of Kullback–Leibler divergence) but much simpler to work with

Model selection & evaluation will be undertaken via cross-validation. By using publicly available voting records,

"one can automatically determine ground truth (and thus avoid the need for manual annotation)"

Getting the Data into Shape

  1. Categorize congressional debates depending on the discussed bills.
  2. Split the data up into individual speech acts
  3. The individual speakers constitue the nodes, and the edges are quotations among between them (notice that we now have an asymmetric adjacency matrix)
  4. Assign label from set $\mathcal{L} = \{approval, disapproval\}$ to each speaker (=node) in a debate using voting history (ex-post)
